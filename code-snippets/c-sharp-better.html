Evolutionary Architecture: Managing Data Mortality

The Challenge: Preserving the integrity of the Greenwald et al. IAT scoring algorithm across 5 years of changing data schemas (V1–V4).
The Solution: A robust inheritance model where modern namespaces (V3) extend legacy logic (V1) to handle new security requirements without breaking historical scoring.

Modern Era

Namespace V3: The Secure Adapter

{
    // New Requirement: Security Tokens

        : base(rsd)
    {
        // Hydrate the new V3 specific data
        _Token = token;
    }
    
    // The Score() method is NOT re-implemented.
    // We implicitly trust the Base (V1) logic.
}
Evolution via Inheritance: Instead of rewriting the parser, V3 strictly adds what is new (The Auth Token) and delegates all mathematical heavy lifting to the battle-tested V1 parent class. This minimizes regression bugs.

The V3 implementation is lightweight. It acts as a wrapper that satisfies modern interface contracts (`IResultSet`) while preserving the historical scoring logic.
